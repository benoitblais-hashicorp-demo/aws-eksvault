# Kubernetes Uptycs EDR Terraform module

This module deploys the Uptycs EDR Kubernetes sensor using Helm and creates the target namespace with stack dependency labels.

## Permissions

Required permissions for the caller identity:

- Kubernetes API permissions to create/update namespaces.
- Helm release management permissions in the target cluster namespace.
- Permission to create and manage release-related Kubernetes objects generated by the chart.

## Authentication

This module uses Kubernetes and Helm provider authentication configured by the caller.

## Features

- Creates a dedicated namespace for the Uptycs EDR deployment.
- Applies stack dependency labels to enforce execution ordering.
- Installs the Uptycs chart with default tag configuration.
- Supports additional values overlays through `additional_values_yaml`.

## Usage example

```hcl
module "k8s_edr_uptycs" {
  source = "./modules/k8s-edr-uptycs"

  addons_dependency_token = "addons-ready"
  cluster_name            = "eks-ca-vso"
  cluster_readiness_token = "rbac-ready"
  uptycs_tags             = "UPDATE/PROD,CCODE/HashiCorp,UT/20A7V,OWNER/security-team@ibm.com"
}
```

## Documentation

## Requirements

The following requirements are needed by this module:

- terraform (~> 1.14)

- helm (~> 2.12)

- kubernetes (~> 2.25)

## Resources

The following resources are used by this module:

- [helm_release.uptycs_edr](https://registry.terraform.io/providers/hashicorp/helm/latest/docs/resources/release) (resource)
- [kubernetes_namespace_v1.edr](https://registry.terraform.io/providers/hashicorp/kubernetes/latest/docs/resources/namespace_v1) (resource)

## Required Inputs

The following input variables are required:

### addons\_dependency\_token

Description: (Required) Dependency token to ensure this component runs after EKS add-ons.

Type: `string`

### cluster\_name

Description: (Required) EKS cluster name used for metadata labeling.

Type: `string`

### cluster\_readiness\_token

Description: (Required) Dependency token to ensure this component runs after cluster RBAC readiness.

Type: `string`

### uptycs\_tags

Description: (Required) Comma-separated Uptycs tags in UPDATE/CCODE/UT/OWNER format.

Type: `string`

## Optional Inputs

The following input variables are optional (have default values):

### additional\_values\_yaml

Description: (Optional) Additional Helm values YAML documents merged after default tag configuration.

Type: `list(string)`

Default: `[]`

### helm\_chart

Description: (Optional) Uptycs Helm chart name.

Type: `string`

Default: `"k8sosquery"`

### helm\_chart\_version

Description: (Optional) Uptycs Helm chart version. Leave empty to install the latest available chart.

Type: `string`

Default: `""`

### helm\_repository

Description: (Optional) Uptycs Helm repository URL.

Type: `string`

Default: `"https://uptycslabs.github.io/kspm-helm-charts"`

### namespace

Description: (Optional) Namespace where Uptycs EDR components are installed.

Type: `string`

Default: `"uptycs"`

## Outputs

The following outputs are exported:

### edr\_helm\_release

Description: Name of the Uptycs EDR Helm release.

### edr\_namespace

Description: Namespace where Uptycs EDR is installed.
